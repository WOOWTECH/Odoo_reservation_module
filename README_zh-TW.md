## 概述

**預約管理系統** 是為 Odoo 18 社群版打造的完整預約管理系統。它將 Odoo 企業版的 Appointment 模組功能帶入社群版 — 線上預約入口、資源管理、付款處理、Discuss 頻道會議室及多語言支援。

### 為什麼選擇此模組？

| 問題痛點 | 解決方案 |
|----------|----------|
| Odoo CE 缺少預約系統 | 完整預約功能，支援 6 種預約類型 |
| 線上預約需要客製開發 | 即用型公開預約入口，路徑為 `/appointment` |
| 會議室需要第三方工具 | 內建 Discuss 頻道整合，支援視訊通話 |
| 資源排程需手動管理 | 自動化的員工、場地、設備可用性管理 |
| 收款流程獨立於預約 | 整合 Odoo 付款供應商，支援按人數收費 |
| 客戶入口無預約管理 | 入口儀表板含預約記錄、詳情及取消功能 |

## 功能特色

### 預約管理

- **6 種預約類型** — 會議、視訊通話、餐桌預訂、資源預訂、付費諮詢、付費座位
- **資源與員工排程** — 管理房間、桌位、設備及員工的每週可用時間
- **自動確認** — 依據容量規則及可用性自動確認預約
- **取消政策** — 可設定取消期限與寬限期
- **FAQ / 問答支援** — 為每種預約類型新增常見問題與解答

### 線上預約入口

- **公開預約頁面** — 精美的 `/appointment` 列表頁面，包含預約類型卡片
- **分步驟預約** — 選擇日期 → 選擇員工 → 選擇時段 → 填寫資料 → 確認
- **響應式設計** — 適配行動裝置的現代化預約流程
- **賓客資訊** — 收集姓名、電子郵件、電話、人數及備註
- **容量驗證** — 跨所有預約強制執行每時段最大容量

### 付款整合

- **Odoo 付款供應商** — 支援所有 Odoo 付款供應商（Stripe、PayPal 等）
- **按人數收費** — 每人價格自動計算總金額
- **付款後確認** — 草稿狀態預約，付款完成後自動確認
- **免費與付費** — 同一系統支援免費和付費預約類型

### Discuss 頻道會議室

- **自動建立頻道** — 確認線上會議預約 → 自動建立 Discuss 頻道
- **視訊通話支援** — 產生會議 URL 並透過郵件分享給客戶
- **入口存取** — 客戶從入口預約頁面直接進入會議室
- **成員管理** — 員工和客戶以正確權限自動加入頻道

### 客戶入口

- **我的預約儀表板** — 入口用戶可查看所有預約的狀態、類型及日期
- **預約詳情頁面** — 完整預約資訊，含「開啟會議室」及「取消預約」操作
- **我的討論** — 從入口存取所有會議室頻道的中央樞紐
- **狀態追蹤** — 即時預約狀態（草稿、已確認、已取消）

### 電子郵件通知

- **確認信件** — 自動發送包含預約詳情、會議連結及日曆附件的郵件
- **提醒系統** — 可設定預約前的郵件提醒
- **會議 URL** — 線上會議預約的所有郵件均包含 Discuss 頻道 URL

### 多語言支援

- **英文** — 完整介面及郵件模板
- **繁體中文 (zh_TW)** — 針對台灣市場的完整翻譯
- **簡體中文 (zh_CN)** — 針對中國市場的完整翻譯
- **可擴展** — 標準 Odoo `.po` 翻譯格式，易於新增語言

### 模組相依圖

```
reservation_module
    │
    ├── calendar     (Odoo 日曆 - 事件管理)
    ├── resource     (資源管理 - 員工與場地排程)
    ├── website      (網站建構器 - 公開預約頁面)
    ├── portal       (客戶入口 - 我的預約儀表板)
    ├── payment      (付款引擎 - Stripe、PayPal 等)

```

## 預約類型

| 類型 | 說明 | 使用情境 |
|------|------|----------|
| **會議** | 與員工預約時間 | 諮詢、面試、一對一會談 |
| **視訊通話** | 含自動建立 Discuss 頻道的虛擬會議 | 遠端諮詢、遠距醫療、線上家教 |

| **資源預訂** | 預訂房間、場地或設備 | 會議室、運動場地、租賃設備 |
| **付費諮詢** | 與專業人士的付費時段 | 法律諮詢、醫療諮詢、教練課程 |
| **付費座位** | 按人數收費的預約 | 活動、工作坊、劇院、團體旅遊 |

## 安裝說明

### 前置需求

- **Odoo 18.0** 社群版
- **PostgreSQL 13+**
- **Python 3.10+**

### 步驟一：部署模組

```bash
# 複製倉庫
git clone https://github.com/WOOWTECH/Odoo_reservation_module.git

# 複製到 Odoo 附加模組路徑
cp -r Odoo_reservation_module/reservation_module /path/to/odoo/addons/

# 更新模組列表並安裝
./odoo-bin -u reservation_module -d your_database
```

### 步驟二：在 Odoo 中安裝

1. 前往 **應用程式** 選單
2. 點擊 **更新應用程式列表**
3. 搜尋 **「Reservation Booking Enhancement」**
4. 點擊 **安裝**

### 步驟三：設定付款（選用）

如需付費預約：

1. 前往 **記帳 > 設定 > 付款供應商**
2. 啟用並設定您偏好的供應商（Stripe、PayPal 等）
3. 模組會自動使用已設定的付款供應商處理付費預約類型

## 設定指南

### 1. 建立預約類型

前往 **行事曆 > 預約**

1. 點擊 **建立** 新增預約類型
2. 設定 **名稱**、**描述** 及 **預約分類**（會議、視訊通話、餐桌、資源、付費諮詢、付費座位）
3. 選擇 **地點類型**：實體、線上會議或客戶地點
4. 指派可處理此預約類型的 **員工**
5. 視需要啟用 **允許客戶選擇員工**

### 2. 設定可用時間

在預約類型表單中，前往 **可用時間** 頁籤：

1. 新增每週可用規則（星期幾 + 時間範圍）
2. 設定 **時段長度**（例如 30 分鐘、1 小時）
3. 設定 **最早/最晚預約** 窗口
4. 設定每時段的 **最大容量**（用於餐桌/資源預訂）

### 3. 設定付款選項

在 **選項** 頁籤：

1. 啟用 **需要付款**
2. 設定 **每人價格**（或固定價格）
3. 連結 **產品** 以產生銷售訂單
4. 設定取消期限

### 4. 設定 Discuss 整合

用於線上會議類型：

1. 將 **地點類型** 設為「線上會議」
2. （選用）設定 **視訊會議連結** 作為會議 URL
3. 系統在預約確認時自動建立 Discuss 頻道
4. 入口用戶自動加入其預約的頻道

### 5. 新增 FAQ / 問答

在 **問題** 頁籤：

1. 新增常見問題及解答
2. 這些會顯示在公開預約頁面

### 6. 發佈

切換 **已發佈** 開關，使預約類型在 `/appointment` 上可見。

## 安全機制

### 存取控制群組

| 群組 | 存取權限 |
|------|----------|
| **公開** | 檢視已發佈的預約類型、提交預約 |
| **入口** | 檢視自己的預約、存取自己的 Discuss 頻道、取消自己的預約 |
| **使用者** | 檢視所有預約、管理自己的預約 |
| **管理員** | 所有預約、預約類型及設定的完整 CRUD |

### 權限模型

```
┌────────────────────────────────────────────────────────┐
│                    存取控制矩陣                          │
│                                                         │
│  appointment.type                                       │
│  ├── 公開:   讀取（僅網站上的已發佈類型）                 │
│  ├── 入口:   讀取                                       │
│  ├── 使用者: 讀取 / 寫入                                 │
│  └── 管理員: 讀取 / 寫入 / 建立 / 刪除                   │
│                                                         │
│  appointment.booking                                    │
│  ├── 公開:   建立（透過網站表單）                         │
│  ├── 入口:   讀取自己的 / 取消自己的                      │
│  ├── 使用者: 讀取全部 / 寫入自己的                        │
│  └── 管理員: 完整 CRUD                                   │
│                                                         │
│  appointment.slot                                       │
│  ├── 公開:   讀取（僅可用時段）                           │
│  ├── 入口:   讀取                                       │
│  ├── 使用者: 讀取 / 寫入                                 │
│  └── 管理員: 完整 CRUD                                   │
│                                                         │
│  resource.resource                                      │
│  ├── 使用者: 讀取                                       │
│  └── 管理員: 完整 CRUD                                   │
└────────────────────────────────────────────────────────┘
```

### 安全特性

- **記錄規則** — 使用者僅能依據其群組檢視/修改預約
- **入口隔離** — 入口用戶僅能看到自己的預約和頻道
- **CSRF 防護** — 所有表單提交均包含 CSRF 令牌
- **付款驗證** — 伺服器端付款驗證，確認付費預約前先驗證付款
- **Discuss 頻道限制** — 僅線上會議預約建立頻道；成員僅限預約相關方
