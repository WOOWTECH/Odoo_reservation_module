# 預約模組 - 使用者指南

## 本模組的功能

**預約模組**是一個自訂的 Odoo 18 模組，為您的 Odoo 系統新增完整的預約和訂位功能。它能讓企業：

- 建立不同類型的預約（會議、諮詢、餐桌訂位、資源預訂）
- 讓客戶透過公開網站自助預約
- 管理資源（會議室、餐桌、場地）及員工的可用時段
- 在預約過程中收集自訂問題
- 接受線上付款的付費預約
- 自動發送確認、取消及提醒電子郵件
- 以行事曆、列表和看板檢視所有預約

本模組同時作為**後端管理工具**（供員工/管理者在 Odoo 介面中使用）及**公開網站**（供客戶瀏覽與預約）。

## 1. 安裝

### 先決條件

- Odoo 18 社群版或企業版
- Python 3.10+
- 必須安裝以下 Odoo 模組：
  - `base`
  - `mail`
  - `calendar`
  - `resource`
  - `website`
  - `payment`（僅在需要線上付款時）

### 安裝步驟

1. **複製模組資料夾**到您的 Odoo 附加模組目錄：
   ```
   將 `reservation_module/` 資料夾複製到 Odoo 的附加模組路徑
   （例如 /opt/odoo/addons/ 或您的自訂附加模組目錄）
   ```

2. **更新附加模組路徑**（如有需要），在 Odoo 設定檔（`odoo.conf`）中：
   ```
   addons_path = /opt/odoo/addons,/path/to/your/custom/addons
   ```

3. **重新啟動 Odoo 伺服器**：
   ```
   sudo systemctl restart odoo
   ```

4. **啟用開發者模式**：
   - 前往**設定**
   - 向下捲動並點選**啟用開發者模式**

5. **更新應用程式列表**：
   - 前往**應用程式**選單
   - 點選**更新應用程式列表**（在頂部選單或搜尋列區域）
   - 確認更新

6. **安裝模組**：
   - 在**應用程式**選單中搜尋 `reservation` 或 `appointment`
   - 找到 **「Reservation Module」**（技術名稱：`reservation_module`）
   - 點選**安裝**

7. 安裝完成後，您將在頂部導覽列看到名為**「預約管理」**的新選單項目。

## 2. 初始設定

安裝完成後，模組即可使用。以下是需要先設定的項目：

### 檢查您的使用者時區

1. 前往**設定 > 使用者與公司 > 使用者**
2. 選擇您的使用者（例如 Administrator）
3. 點選**偏好設定**頁籤
4. 確認**時區**正確（例如 Asia/Taipei）
5. 此設定會影響時段如何顯示給客戶

> **注意：** 在 Odoo 18 中，時區是每位使用者的個別設定，位於使用者偏好設定中，而非一般設定或公司設定。

### 了解使用者角色

本模組建立兩個安全群組：

| 角色 | 適用對象 | 可執行的操作 |
|------|---------|-------------|
| **預約使用者** | 所有內部員工 | 檢視預約類型、檢視/建立指派給自己的預約 |
| **預約管理員** | 管理者、預約主管 | 完全控制：建立/編輯/刪除所有類型、預約和設定 |

指派角色的方式：
1. 前往**設定 > 使用者與公司 > 使用者**
2. 選擇使用者
3. 在**其他**區段或自訂群組中，指派**預約使用者**或**預約管理員**

## 3. 建立預約類型

預約類型是本模組的核心。每種類型定義了客戶可進行的特定預約種類。

### 前往預約類型

1. 點選頂部選單中的**預約管理**
2. 點選**預約類型**

### 建立新的預約類型

1. 點選**新增**
2. 填寫基本資訊：

#### 基本資訊頁籤

| 欄位 | 說明 | 範例 |
|------|------|------|
| **名稱** | 客戶會看到的顯示名稱 | 「一小時諮詢」 |
| **分類** | 預約的類型 | 選項：會議、視訊通話、餐桌訂位、資源預訂、付費諮詢、付費座位 |
| **啟用** | 此類型是否可用 | 勾選以啟用 |
| **已發佈** | 是否在網站上顯示 | 勾選以公開顯示 |
| **時段時長** | 每次預約的長度（小時） | 1.0 = 1 小時、0.5 = 30 分鐘 |
| **時區** | 排程使用的時區 | 例如 Asia/Taipei |

#### 預約類型與指派方式

| 欄位 | 說明 |
|------|------|
| **預約類型** | **使用者** = 以員工為基準（客戶預約特定人員）、**資源** = 以資源為基準（客戶預約會議室/餐桌/場地） |
| **指派方式** | **自動** = 系統挑選最佳可用員工/資源、**客戶選擇** = 客戶自行選擇員工/資源 |

#### 地點頁籤

| 欄位 | 說明 |
|------|------|
| **地點類型** | **線上**或**實體** |
| **視訊連結** | 視訊會議的 URL（線上時使用） |
| **地點** | 選擇聯絡人/合作夥伴作為實體地址 |

#### 排程選項（在選項頁籤中）

| 欄位 | 說明 | 預設值 |
|------|------|--------|
| **最大預約天數** | 客戶最多可提前幾天預約 | 30 |
| **最短預約提前時數** | 最少提前通知時間（小時） | 1.0 |
| **取消截止時數** | 預約前多少小時內可取消 | 1.0 |
| **自動確認** | 自動確認預約（相對於手動審核） | 是 |
| **時段間隔** | 可用開始時間之間的時間間隔 | 1.0 小時 |

#### 容量設定（在選項頁籤中）

| 欄位 | 說明 |
|------|------|
| **管理容量** | 當一個時段可服務多位客人時啟用（例如餐桌） |
| **最大同時預約數** | 一個時段可重疊的預約數量 |

### 預設分類說明

| 分類 | 適用場景 | 範例 |
|------|---------|------|
| **會議** | 與員工的現場會議 | 商務諮詢 |
| **視訊通話** | 遠端會議 | 線上諮商 |
| **餐桌訂位** | 餐廳/咖啡廳訂位 | 晚餐訂位 |
| **資源預訂** | 設施/設備租借 | 網球場、會議室 |
| **付費諮詢** | 付費專業服務 | 法律諮詢 |
| **付費座位** | 需付費的活動座位 | 研討會座位 |

## 4. 管理資源

資源代表可預訂的實體物品：會議室、餐桌、場地、設備等。

### 新增資源

1. 編輯預約類型時，前往**可用時段**頁籤
2. 在**資源**區段中，點選**新增一行**
3. 選擇現有資源或建立新資源

### 建立新資源

新增資源時，點選**建立並編輯**以設定新資源：

| 欄位 | 說明 | 範例 |
|------|------|------|
| **名稱** | 資源名稱 | 「會議室 A」 |
| **容量** | 可容納人數 | 10 |
| **圖示** | Font Awesome 圖示類別 | "fa-building" |
| **圖片** | 資源的照片 | 上傳照片 |

### 指派員工

對於以員工為基準的預約類型：
1. 在**可用時段**頁籤中，找到**員工**區段
2. 點選**新增一行**以指派可處理此預約類型的使用者

## 5. 設定可用時段

可用時段定義了預約可被預訂的時間。

### 每週循環排程

1. 開啟預約類型
2. 前往**可用時段**頁籤
3. 將**排程類型**設為**循環**
4. 在可用時段表格中，為每個工作時段新增列：

| 欄位 | 說明 |
|------|------|
| **星期幾** | 週一到週日 |
| **開始** | 開始時間（例如 9.0 = 上午 9:00） |
| **結束** | 結束時間（例如 17.0 = 下午 5:00） |
| **資源** | （選填）此時段適用的特定資源 |
| **員工** | （選填）此時段適用的特定員工 |

**範例：平日供應晚餐、週末全天營業的餐廳：**

| 星期 | 開始 | 結束 |
|------|------|------|
| 週一 | 18.0 | 22.0 |
| 週二 | 18.0 | 22.0 |
| 週三 | 18.0 | 22.0 |
| 週四 | 18.0 | 22.0 |
| 週五 | 18.0 | 22.0 |
| 週六 | 12.0 | 23.0 |
| 週日 | 12.0 | 21.0 |

### 提示

- 時間使用 24 小時制的小數格式：上午 9:30 = 9.5、下午 2:00 = 14.0
- 同一天可新增多列（例如上午和下午時段，中間休息）
- 如果在某列指定了資源或員工，該可用時段僅適用於他們
- 留空資源/員工則該可用時段適用於所有人

## 6. 新增自訂問題

您可以在預約過程中向客戶收集額外資訊。

### 新增問題

1. 開啟預約類型
2. 前往**問題**頁籤
3. 點選**新增一行**

### 問題欄位

| 欄位 | 說明 |
|------|------|
| **問題文字** | 顯示給客戶的問題 |
| **類型** | text、textarea、select、radio、checkbox、date、datetime、number、email、phone |
| **必填** | 客戶是否必須回答 |
| **佔位文字** | 輸入欄位內的提示文字 |
| **說明文字** | 顯示在問題下方的額外說明 |

### 問題類型說明

| 類型 | 用途 | 範例 |
|------|------|------|
| **text** | 簡短文字回答 | 「您的公司名稱」 |
| **textarea** | 長篇文字回答 | 「描述您的專案需求」 |
| **select** | 下拉式選單，單選 | 「經驗程度」（初學者/中級/進階） |
| **radio** | 單選按鈕，單選 | 「偏好的會議方式」 |
| **checkbox** | 可多選 | 「飲食需求」（素食、純素、無麩質） |
| **date** | 日期選擇器 | 「偏好的替代日期」 |
| **number** | 數字輸入 | 「額外賓客人數」 |
| **email** | 含驗證的電子郵件輸入 | 「額外聯絡電子郵件」 |
| **phone** | 電話號碼輸入 | 「緊急聯絡電話」 |

### 新增選項（適用於 select、radio、checkbox 類型）

1. 建立 select/radio/checkbox 問題後，點選該問題行
2. 在**選項**區段中，點選**新增一行**
3. 輸入每個選項名稱（例如「素食」、「純素」、「無麩質」）

## 7. 付款設定

對於付費預約類型（諮詢、預訂座位），您可以要求線上付款。

### 先決條件

1. 在 Odoo 中安裝並設定**付款**模組
2. 設定至少一個付款供應商（例如 Stripe、PayPal）
   - 前往**會計 > 設定 > 付款供應商**
   - 啟用並設定您偏好的供應商

### 為預約類型啟用付款

1. 開啟預約類型
2. 前往**選項**頁籤
3. 找到**付款**區段：

| 欄位 | 說明 |
|------|------|
| **需要付款** | 勾選以要求確認前付款 |
| **付款金額** | 收取的價格 |
| **按人收費** | 勾選後，金額會乘以賓客人數 |
| **付款產品** | 連結至產品記錄（用於發票/會計） |
| **幣別** | 付款使用的貨幣 |

### 付款流程

1. 客戶完成預約表單
2. 重新導向至付款頁面
3. 選擇付款方式並完成交易
4. 付款成功時：
   - 預約自動確認
   - 付款狀態變更為「已付款」
   - 發送確認電子郵件
5. 付款失敗時，預約維持「草稿」狀態，付款狀態為「待處理」

## 8. 發佈到網站

### 讓預約類型可見

1. 開啟預約類型
2. 確認**已發佈**（is_published）已勾選
3. 預約將顯示在您的網站：`https://your-domain.com/appointment`

### 網站網址

| 網址 | 顯示內容 |
|------|---------|
| `/appointment` | 所有已發佈預約類型的列表 |
| `/appointment/<id>` | 特定預約類型的詳細頁面 |
| `/appointment/<id>/schedule` | 行事曆與時段選擇頁面 |
| `/appointment/<id>/book` | 預約表單 |

### 分享連結

1. 開啟預約類型
2. 點選表單標題中的**分享**按鈕
3. 複製產生的網址分享給客戶

## 9. 客戶預約流程

以下是客戶的逐步體驗：

### 步驟 1：瀏覽可用預約

- 客戶造訪您網站的 `/appointment`
- 看到預約類型的卡片網格，每張卡片顯示名稱、時長和圖示/圖片

### 步驟 2：選擇預約類型

- 客戶點選預約類型卡片
- 看到說明、地點資訊和「立即預約」按鈕

### 步驟 3：選擇日期和時間

- 客戶看到顯示當月的**行事曆**
- 可預約的日期會醒目標示；不可預約的日期會呈灰色
- 點選日期以查看可用時段
- 如果預約類型使用**客戶選擇**指派方式：
  - 可以選擇想要的資源或員工
- 點選可用的時段

### 步驟 4：填寫預約資訊

- 客戶輸入個人資訊：
  - **姓名**（必填）
  - **電子郵件**（必填）
  - **電話**（選填）
  - **賓客人數**（如有管理容量）
  - **備註**（選填）
- 回答為此預約類型設定的自訂問題
- 點選**確認預約**

### 步驟 5：付款（如有要求）

- 如果預約類型需要付款，客戶會被重新導向至付款頁面
- 選擇付款方式並完成交易

### 步驟 6：確認

- 客戶看到確認頁面，包含：
  - 預約參考編號（例如 APT00001）
  - 日期、時間和時長
  - 地點或視訊連結
  - 檢視或管理預約的連結
- 同時會發送確認電子郵件至客戶的電子郵件地址

### 步驟 7：管理預約

- 客戶可透過確認郵件中的連結存取預約
- 可執行以下操作：
  - 檢視預約詳情
  - 取消預約（如在取消截止時間內）

## 10. 管理預約

### 檢視所有預約

1. 前往**預約管理 > 所有預約**
2. 以列表檢視查看預約的關鍵資訊

### 預約狀態流程

```
草稿 → 已確認 → 已完成
  ↓         ↓
  └→ 已取消 ←┘
```

| 狀態 | 意義 |
|------|------|
| **草稿** | 新建立，尚未確認 |
| **已確認** | 已核准並已排程 |
| **已完成** | 預約已完成 |
| **已取消** | 預約已取消 |

### 預約操作

開啟任何預約以查看詳情並執行操作：

| 按鈕 | 功能 | 何時可用 |
|------|------|---------|
| **確認** | 確認預約並建立行事曆事件 | 草稿狀態時 |
| **標記完成** | 將預約標記為已完成 | 已確認狀態時 |
| **取消** | 取消預約 | 草稿或已確認狀態時 |
| **重設為草稿** | 將已取消的預約恢復為草稿 | 已取消狀態時 |

### 預約詳情

每筆預約記錄包含：

- **預約參考編號**：自動產生（APT00001、APT00002 等）
- **預約類型**：哪種類型的預約
- **日期與時間**：開始和結束日期時間
- **時長**：自動計算
- **賓客資訊**：姓名、電子郵件、電話、賓客人數
- **資源/員工**：指派的資源或員工
- **付款狀態**：不需要 / 待處理 / 已付款 / 已退款
- **回答頁籤**：客戶對自訂問題的回覆
- **備註頁籤**：客戶備註和內部員工備註
- **對話記錄**：完整的訊息歷史和活動追蹤

### 篩選和搜尋預約

使用搜尋列和篩選條件來尋找預約：

- **快速篩選**：今天、本週、即將到來
- **狀態篩選**：草稿、已確認、已完成、已取消
- **分組依據**：預約類型、狀態、日期、資源、員工

## 11. 行事曆檢視

本模組提供多種行事曆檢視以進行視覺化排程：

### 所有預約行事曆

1. 前往**預約管理 > 所有預約**
2. 切換至**行事曆檢視**（點選右上角檢視切換器中的行事曆圖示）
3. 預約依預約類型以不同顏色標示

### 資源行事曆

1. 前往**預約管理 > 資源預約**
2. 依資源檢視預約
3. 依資源以不同顏色標示，方便辨識

### 員工行事曆

1. 前往**預約管理 > 員工預約**
2. 依員工檢視預約
3. 依員工以不同顏色標示，方便辨識

### 行事曆功能

- **日/週/月檢視**：使用頂部按鈕切換不同檢視
- **點選建立**：點選空白時段以建立新預約
- **拖曳重新排程**：將預約拖曳至不同時間（需要編輯權限）
- **點選檢視**：點選任何預約以查看詳情

## 12. 安全性與使用者角色

### 角色權限摘要

| 操作 | 公開（網站） | 預約使用者 | 預約管理員 |
|------|-------------|-----------|-----------|
| 檢視已發佈的預約類型 | 是 | 是 | 是 |
| 檢視所有預約類型 | 否 | 是 | 是 |
| 建立/編輯/刪除預約類型 | 否 | 否 | 是 |
| 檢視自己的預約 | 透過存取令牌 | 是 | 是 |

| 建立預約 | 是（透過網站） | 是 | 是 |

| 存取報表與設定選單 | 否 | 否 | 是 |

### 存取令牌

- 此令牌允許客戶在不登入的情況下檢視和管理其特定預約
- 令牌是唯一且安全的

## 15. 疑難排解

### 常見問題

**問：我安裝了模組但看不到選單。**
- 確認您登入的使用者具有**預約使用者**或**預約管理員**角色
- 嘗試清除瀏覽器快取並重新整理頁面

**問：客戶在網站上看不到任何預約類型。**
- 檢查預約類型是否已勾選**已發佈**
- 確認已安裝**網站**模組
- 確認已設定可用時段（沒有可用時段 = 沒有時段 = 無法預約）

**問：客戶看不到任何時段。**
- 確認預約類型中已設定**可用時段**列
- 檢查預約日期是否在**最大預約天數**範圍內
- 確認預約時間符合**最短預約提前時數**的要求
- 如果使用資源/員工，確認它們已指派給該預約類型

**問：付款功能無法使用。**
- 確認已安裝**付款**模組
- 設定至少一個付款供應商（設定 > 付款供應商）
- 啟用付款供應商並將其設為「正式」或「測試」模式
- 檢查預約類型是否已啟用**需要付款**並設定了**金額**

**問：電子郵件沒有發送。**
- 檢查 Odoo 外寄郵件伺服器設定（設定 > 技術 > 外寄郵件伺服器）
- 確認郵件範本存在且為啟用狀態
- 檢查郵件佇列（設定 > 技術 > 電子郵件 > 郵件）是否有錯誤

**問：確認預約時未建立行事曆事件。**
- 確認已安裝**行事曆**模組
- 行事曆事件會在預約確認時自動建立

**問：我需要變更語言。**
- 本模組支援英文、繁體中文（zh_TW）和簡體中文（zh_CN）
- 前往**設定 > 翻譯 > 載入翻譯**並選擇您的語言
- 模組會自動使用使用者的語言偏好設定
